Claude Code en action — regarde, il code tout seul

Tu connais ChatGPT ?
Ok. Oublie.

ChatGPT c'est bien. Mais c'est un chatbot — tu poses une question, il te donne du texte. Après c'est à toi de copier-coller, créer les fichiers, tout faire à la main. Claude Code c'est pas ça. C'est un agent. Il agit sur ton ordinateur. Tu lui dis "crée-moi un site", il le fait. Pour de vrai.

ChatGPT

→ Tu poses une question

→ Il te donne du texte

→ Tu copies le code

→ Tu crées les fichiers toi-même

→ Tu colles le code dedans

→ Tu testes, ça marche pas

→ Tu retournes demander

Claude Code

→ Tu dis "crée un site web"

→ Il crée les fichiers

→ Il écrit le code dedans

→ Il teste

→ Tu vois le résultat

→ Tu dis "change le titre"

→ C'est fait

Ce que ça change pour toi

Tu sais pas coder ? C'est pas grave. Claude Code code pour toi. Toi tu donnes la direction, lui il construit. C'est un dialogue — tu décris ce que tu veux, il le fait, tu ajustes. Comme si t'avais un développeur à côté de toi H24. Et c'est cet outil que tu vas utiliser dans presque toutes les séances de la formation.

Comment bien lui parler.

C'est la compétence la plus importante. Mieux tu lui parles, meilleur est le résultat. 3 règles simples.

Règle 1 — Sois précis

Vague

"Fais-moi un site"

Il sait pas ce que tu veux. Résultat générique.

Précis

"Crée un site vitrine pour un plombier à Lyon. Page d'accueil avec un hero, services avec 4 cartes, formulaire de contact."

Il sait exactement quoi faire.

Règle 2 — Donne du contexte

Dis-lui pour qui, pour quoi, quel style. Plus il en sait, mieux c'est.

> Crée un script qui scrape les restaurants de Lyon sur Google Maps. Je veux le nom, l'adresse, le téléphone et la note. Exporte en CSV. Code simple et commenté parce que je débute.

Tu vois ? Quoi, d'où, quoi récupérer, quel format, et ton niveau. Claude Code te livre exactement ça.

Règle 3 — Itère

Tu vas pas avoir le résultat parfait du premier coup. Et c'est normal. Le process :

1

Tu donnes une instruction

2

Tu regardes le résultat

3

Tu corriges : "change ça", "ajoute ça", "enlève ça"

4

Tu re-regardes

5

Quand c'est bon, tu passes à la suite

Le fichier CLAUDE.md — le briefing permanent

C'est un fichier texte que tu mets dans ton projet. Claude Code le lit automatiquement à chaque session. Il sait qui tu es, ce que tu fais, comment tu veux bosser. Plus besoin de répéter le contexte.

// Exemple de CLAUDE.md
# Mon projet

## Contexte
Site vitrine pour mon activité d'Expert IA freelance.

## Stack
HTML / Tailwind CSS / WordPress

## Règles
- Code simple et commenté
- Design moderne, fond blanc
- Mobile-first
- SEO optimisé

Tu le crées une fois. Ou tape /init et Claude Code le génère pour toi.

Les limites — soyons honnêtes.

Claude Code c'est puissant mais c'est pas magique. Voilà ce qu'il faut savoir.

Il peut se tromper

Vérifie toujours le résultat. Parfois le code marche pas, parfois il fait pas exactement ce que tu voulais. C'est normal — tu corriges et tu avances.

Il voit que ton dossier

Claude Code voit les fichiers du dossier où tu es. Si t'es dans le mauvais dossier, il voit pas ton projet.

L'astuce : découpe en petites étapes

Au lieu de "crée un site complet avec 10 pages", dis "crée la page d'accueil", puis "crée la page contact", etc. Plus c'est découpé, meilleur c'est.
