Séance 1 sur 24

Claude Code.
Ton outil principal.

C'est l'IA dans ton terminal. Tu lui parles, il construit. Pas besoin de savoir coder. Dans cette séance, tu apprends ce que c'est, comment l'installer, et comment t'en servir.

35-45 min de vidéo
1
Partie 1

C'est quoi Claude Code ?

En une phrase : c'est une IA qui vit dans ton terminal et qui peut agir sur ton ordinateur. Tu lui dis ce que tu veux, elle le fait.

C'est pas un chatbot.

ChatGPT, tu lui poses une question, il te donne du texte. C'est bien. Mais c'est toi qui dois copier-coller, créer les fichiers, tout faire à la main.

Claude Code c'est différent. Tu lui dis "crée-moi un site web" et il le fait. Vraiment. Il crée les fichiers, écrit le code, structure le projet. Toi tu regardes et tu ajustes.

La différence concrètement

ChatGPT

→ Tu poses une question

→ Il te donne du texte

→ Tu copies le code

→ Tu crées les fichiers toi-même

→ Tu colles le code dedans

→ Tu testes, ça marche pas

→ Tu retournes demander

Claude Code

→ Tu dis "crée un site web"

→ Il crée les fichiers

→ Il écrit le code dedans

→ Il teste

→ Tu vois le résultat

→ Tu dis "change le titre"

→ C'est fait

Ce que ça change pour toi

Tu sais pas coder ? C'est pas grave. Claude Code code pour toi. Toi tu donnes la direction, lui il construit. C'est un dialogue — tu décris ce que tu veux, il le fait, tu ajustes. Comme si t'avais un développeur à côté de toi H24.

2
Partie 2

Installer Claude Code.

C'est rapide. 5 minutes max. On fait ça étape par étape.

Étape 1 — Installer Node.js

Claude Code tourne avec Node.js. C'est un outil que ton ordinateur a besoin pour exécuter Claude Code. Si tu l'as déjà, passe à l'étape suivante.

Sur Mac

Ouvre le Terminal (cherche "Terminal" dans Spotlight) et tape :

brew install node

Si t'as pas Homebrew, va sur nodejs.org et télécharge l'installeur.

Sur Windows

Va sur nodejs.org, télécharge l'installeur, double-clique, suivant-suivant-terminer.

Pour vérifier que c'est installé :

node --version

Si tu vois un numéro (genre v20.11.0), c'est bon.

Étape 2 — Le terminal, c'est quoi ?

C'est une fenêtre où tu tapes des commandes. Pas de boutons, pas de menus — juste du texte. C'est là que Claude Code vit.

Sur Mac

Cherche "Terminal" dans Spotlight (Cmd + Espace). Ou va dans Applications → Utilitaires → Terminal.

Sur Windows

Cherche "PowerShell" ou "Terminal" dans la barre de recherche. Clique droit → "Exécuter en administrateur".

T'inquiète pas si ça fait peur au début. Tout le monde a eu cette réaction. En 10 minutes tu seras à l'aise.

Étape 3 — Installer Claude Code

Une seule commande :

npm install -g @anthropic-ai/claude-code

Ça prend 30 secondes. npm va télécharger Claude Code et l'installer sur ton ordinateur.

Pour vérifier que c'est installé :

claude --version

Tu vois un numéro ? C'est bon, Claude Code est prêt.

Étape 4 — Premier lancement

Tape juste :

claude

La première fois, il va te demander de te connecter avec ton compte Anthropic. Tu suis les instructions à l'écran — ça prend 1 minute.

Si t'as pas de compte Anthropic

Va sur console.anthropic.com, crée un compte, ajoute un moyen de paiement. Les premiers crédits suffisent largement pour commencer.

// Récap — Installation en 4 commandes
1. node --version → vérifier que Node.js est là
2. npm install -g @anthropic-ai/claude-code → installer
3. claude --version → vérifier que c'est installé
4. claude → lancer

C'est tout. 5 minutes. Tu passes de "j'ai rien" à "Claude Code est prêt".

C'est ça. Un curseur qui attend tes instructions. Tu tapes en français, il comprend et il agit.

C'est un dialogue

C'est pas un formulaire où tu remplis des champs. C'est une conversation. Tu parles, il fait, tu corriges, il ajuste. Comme avec un humain.

> Crée un fichier HTML avec un titre "Mon Site"

✓ Fichier créé : index.html

> Change le titre en "Expert IA"

✓ Titre modifié dans index.html

> Ajoute une section "À propos" avec 3 paragraphes

✓ Section ajoutée dans index.html

Tu vois ? Tu décris → il fait → tu ajustes → il corrige. C'est fluide.

Les fichiers qu'il crée

Quand Claude Code crée des fichiers, ils apparaissent dans le dossier où tu es. Par exemple, si tu es dans ~/projets/mon-site, les fichiers seront là.

// Exemple de structure créée
mon-site/
├── index.html
├── styles.css
├── script.js
└── images/

Tu peux les ouvrir dans ton navigateur, les modifier, les envoyer à un client. C'est des vrais fichiers.

Il voit ce qui est autour de lui

Claude Code voit tous les fichiers du dossier où tu es. Si t'as déjà un projet avec du code, il le lit et il comprend. Tu peux lui dire :

Lis le fichier index.html et dis-moi ce qu'il contient
Modifie le header pour ajouter un menu de navigation
Corrige le bug dans le formulaire de contact

Il a le contexte. Il voit ton projet. Il sait ce qu'il y a dedans.

4
Partie 4

Comment lui parler.

C'est la compétence la plus importante. Mieux tu lui parles, meilleur est le résultat.

Règle n°1 — Sois précis

Vague (mauvais)

"Fais-moi un site"

Claude Code sait pas ce que tu veux. Il va faire un truc générique.

Précis (bien)

"Crée un site vitrine pour un plombier à Lyon. Page d'accueil avec un hero, une section services avec 4 cartes, et un formulaire de contact."

Claude Code sait exactement quoi faire.

Règle n°2 — Donne du contexte

Dis-lui pour qui, pour quoi, quel style. Plus il en sait, mieux c'est.

> Crée un script qui scrape les restaurants de Lyon sur Google Maps. Je veux le nom, l'adresse, le téléphone et la note. Exporte en CSV. Le style de code doit être simple et commenté parce que je débute.

Tu vois ? T'as dit quoi (scraper des restos), d'où (Google Maps Lyon), quoi récupérer (nom, adresse, tel, note), quel format (CSV), et ton niveau (débutant). Claude Code va te livrer exactement ça.

Règle n°3 — Itère

Tu vas pas avoir le résultat parfait du premier coup. Et c'est normal. Le process c'est :

1

Tu donnes une instruction

2

Tu regardes le résultat

3

Tu corriges : "change ça", "ajoute ça", "enlève ça"

4

Tu re-regardes

5

Quand c'est bon, tu passes à la suite

C'est comme travailler avec quelqu'un. Tu guides, il exécute, tu ajustes. Ça va vite.

Le fichier CLAUDE.md

C'est un fichier texte que tu mets à la racine de ton projet. Claude Code le lit automatiquement à chaque fois. C'est son "briefing" — il sait qui tu es, ce que tu fais, comment tu veux qu'il travaille.

// Exemple de CLAUDE.md
# Mon projet

## Contexte
Je suis Expert IA freelance. Ce projet est un site
vitrine pour mon activité.

## Stack
- HTML / Tailwind CSS
- Déploiement sur WordPress

## Règles
- Code simple et commenté
- Design moderne, fond blanc
- Mobile-first
- SEO optimisé dès le départ

Tu le crées une fois, Claude Code le lit à chaque session. Plus besoin de répéter le contexte.

Comment le créer

Tu peux même demander à Claude Code de le faire :

Crée un fichier CLAUDE.md pour ce projet. C'est un site vitrine pour un expert IA freelance. Stack : HTML + Tailwind. Design moderne, fond blanc, mobile-first.

Ou utilise la commande :

/init

Claude Code analyse ton projet et génère un CLAUDE.md automatiquement.

Partie 5

Les commandes essentielles.

Y'en a pas 50. Voilà celles que tu vas utiliser tous les jours.

Les slash commands

Dans Claude Code, les commandes qui commencent par / sont des raccourcis :

/help Voir toutes les commandes disponibles
/clear Repartir de zéro — effacer la conversation
/cost Voir combien t'as dépensé en tokens
/compact Compresser la conversation quand elle devient longue
/init Générer un CLAUDE.md automatiquement

Gérer les fichiers

Tu parles en français, Claude Code comprend :

Lis le fichier index.html
Modifie le titre dans index.html
Crée un nouveau fichier contact.html
Supprime le fichier test.js
Montre-moi la structure du projet

Pas de syntaxe spéciale. Tu parles normalement.

Exécuter des commandes

Claude Code peut aussi lancer des commandes système :

Lance un serveur local pour voir le site
Installe la librairie tailwindcss
Fais un git commit avec le message "première version"

Il te demandera confirmation avant d'exécuter. Tu valides, il lance.

Slash commands → /help, /clear, /cost, /compact, /init
Fichiers → "lis", "modifie", "crée", "supprime"
Commandes → "lance", "installe", "git commit"
Contexte → CLAUDE.md à la racine du projet
Dialogue → tu parles en français, il comprend

Partie 7

Les limites à connaître.

Claude Code c'est puissant mais c'est pas magique. Voilà ce qu'il faut savoir.

Il peut se tromper

Vérifie toujours le résultat. Parfois le code marche pas, parfois il fait pas exactement ce que tu voulais. C'est normal — tu corriges et tu avances.

Il voit que ton dossier

Claude Code voit les fichiers du dossier où tu es. Si t'es dans le mauvais dossier, il voit pas ton projet. Pense à te placer au bon endroit avant de lancer.

Ça coûte des tokens

Chaque échange consomme des tokens (= de l'argent). Les conversations longues coûtent plus cher. Utilise /cost pour surveiller et /compact pour compresser quand ça devient long.

L'astuce : découpe en petites étapes

Au lieu de dire "crée-moi un site complet avec 10 pages", dis "crée la page d'accueil", puis "maintenant crée la page contact", etc. Plus c'est découpé, meilleur c'est le résultat.

EXEMPLES CONCRETS

Des exemples pour bien comprendre.

Voilà des vraies instructions que tu pourrais donner à Claude Code. Pour que tu vois le genre de choses qu'il sait faire.

Exemple 1 — Créer un site
Crée un site vitrine en HTML + Tailwind pour un consultant IA. Design moderne, fond blanc, 4 sections : hero, services, témoignages, contact. Mobile-first.

Claude Code crée tous les fichiers, le design, le responsive. En 5 minutes t'as un site.

Exemple 2 — Scraper des données
Écris un script Python qui scrape les 100 premiers résultats Google pour "cabinet comptable Lyon". Je veux le nom, le site web et le téléphone. Exporte en CSV.

Claude Code écrit le script, l'exécute, et te sort le fichier CSV avec les données.

Exemple 3 — Modifier un projet existant
Lis tout le projet et ajoute un formulaire de contact qui envoie un email quand quelqu'un le remplit. Utilise un style cohérent avec le reste du site.

Claude Code lit les fichiers existants, comprend le style, et ajoute le formulaire qui matche.

Exemple 4 — Générer du contenu
Génère 10 articles de blog SEO sur le thème "IA pour PME". Chaque article fait 800 mots, avec un titre accrocheur, des sous-titres, et une meta description. Sauvegarde chaque article dans un fichier séparé.

Claude Code crée 10 fichiers, chacun avec un article complet, structuré, optimisé SEO.

Exemple 5 — Corriger un bug
Le formulaire de contact envoie pas les emails. Regarde le code et corrige le problème.

Claude Code lit le code, trouve le bug, le corrige, et t'explique ce qu'il a changé.
